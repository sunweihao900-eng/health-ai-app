# 数据安全措施说明

**健康AI咨询助手**
版本：1.0 | 2026年3月20日

---

## 一、安全架构概述

本服务采用纵深防御安全架构，覆盖传输、存储、访问控制、PII保护四个核心层面。

```
用户 ──[HTTPS/TLS 1.3]──► API网关 ──► FastAPI后端
                                          │
                              ┌───────────┼───────────┐
                              ▼           ▼           ▼
                          PII脱敏     加密存储    访问控制
                              │           │           │
                         (传输前)    (AES-256)   (JWT+RBAC)
```

## 二、传输安全

### 2.1 加密传输
- 全程使用 **HTTPS（TLS 1.3）** 加密
- 禁止HTTP明文传输（HSTS强制执行）
- 证书使用受信任CA签发，有效期监控自动告警

### 2.2 API安全
- 所有接口需JWT认证（HS256签名，1小时过期）
- 接口限流：10次/分钟/用户（防止暴力请求）
- CORS严格白名单控制
- API网关层防DDoS保护

## 三、存储安全

### 3.1 数据加密
- 静止数据：**AES-256** 加密存储
- 数据库：加密连接，访问需证书认证
- 备份文件：独立密钥加密

### 3.2 数据分类存储
| 数据分类 | 存储方式 | 访问级别 |
|---------|---------|---------|
| 脱敏对话日志 | 加密数据库 | 仅安全团队 |
| 用户账户信息 | 加密数据库（密码bcrypt哈希） | 仅认证系统 |
| 系统日志 | 结构化日志存储 | 运维团队 |
| 临时缓存 | 内存（不落磁盘） | 仅当前会话 |

### 3.3 数据生命周期
- 对话日志：**90天后自动清除**
- 账户数据：注销后30天内删除
- 日志数据：6个月后归档清除
- 临时数据：会话结束即销毁

## 四、PII（个人信息）保护

### 4.1 脱敏实施
所有用户输入在进入后端处理前**强制执行PII脱敏**：

```python
# 脱敏规则示例
手机号:  138****5678   # 保留前3后4
身份证:  110101*******1  # 仅保留地区码
邮箱:    ab***@example.com
银行卡:  **** **** **** 1234
姓名:    李**
```

### 4.2 脱敏覆盖范围
- ✅ 手机号（1开头11位）
- ✅ 身份证号（18位）
- ✅ 电子邮箱
- ✅ 银行卡号（16-19位）
- ✅ 家庭地址（前缀识别）
- ✅ 中文姓名（基于前缀关键词）

### 4.3 脱敏记录
系统记录每次脱敏操作的类型（不记录原始值），用于合规审计。

## 五、访问控制

### 5.1 用户认证
- JWT Token认证，签名密钥定期轮换
- 生产环境使用强密钥（256位随机数）
- Token有效期1小时，续签需重新认证

### 5.2 服务器访问
- 服务器禁止密码登录，仅允许SSH密钥认证
- 堡垒机跳转，所有操作记录审计日志
- 最小权限原则，按职能分配权限

### 5.3 数据库访问
- 数据库仅接受来自应用服务器的连接（内网隔离）
- 定期审查数据库账号权限
- 禁止直连生产数据库进行数据查询
